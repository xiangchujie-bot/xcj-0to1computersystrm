🔧 第二章：ALU的11个核心组件

1️⃣ 右移器

把二进制数向右移动
例：0001 1001 → 1000 1100

2️⃣ 左移器

把二进制数向左移动
例：0100 0010 → 1000 0100

3️⃣ 反相器

每一位取反（0变1，1变0）
例：0110 1000 → 1001 0111

4️⃣ 与运算器

两个数对应位都是1才输出1
用途：提取特定位、掩码操作

5️⃣ 或运算器

两个数对应位有1就输出1
用途：合并数据、设置特定位

6️⃣ 异或运算器

两个数对应位不同才输出1
用途：数据加密、检错

7️⃣ 加法器

实现二进制加法
包含进位处理机制

8️⃣ 比较器

判断两个数的大小关系
输出：相等、大于、小于

9️⃣ 数值0

生成全0的字节
用于清零、初始化

🔟 数值1

生成数值1的字节
用于计数、递增

1️⃣1️⃣ 全0判断

判断整个字节是否全为0
用于条件判断

💡 重要提示

这11个组件都是通过逻辑门（与门、或门、非门、异或门）的不同组合方式构建的！

虽然看起来功能各异，但底层原理都是一样的——基于0和1的电信号处理。

🎙️ 第五章：三态门使能 - 数字电路的"交通警察"

🤔 什么是三态门？

三态门（Tri-state Gate）是一种特殊的逻辑门，它有三种输出状态：

1️⃣
高电平（1）
信号输出为1
0️⃣
低电平（0）
信号输出为0
⚡
高阻态（Z）
断开连接状态

🎤 形象比喻：演讲台的麦克风

想象一个演讲台上有一个麦克风，多人轮流发言：

👤 说话的人（使能信号=1）：
麦克风处于工作状态，输入什么声音就输出什么声音

🙊 听众（使能信号=0）：
麦克风处于高阻态，即使说话也不会被广播出去
相当于麦克风被静音了，完全断开

🎤
使能=1时：
输入 → 输出 ✅

使能=0时：
输入 → 高阻态 ⚡
（完全断开，不影响总线）

🚦 为什么需要三态门？

🚌

总线共享

问题：多个设备共用一条数据总线

如果多个设备同时输出信号，会发生冲突！

解决：

✓ 只允许一个设备的三态门使能=1

✓ 其他设备的三态门使能=0（高阻态）

✓ 这样就不会互相干扰

🎛️

寄存器控制

场景：CPU有多个寄存器

R0、R1、R2、R3都连到同一条总线上

通过使能信号控制：

• 哪个寄存器输出数据到总线

• 哪些寄存器保持高阻态

💡 避免多个寄存器同时输出造成冲突

🔀

多设备控制

核心作用：实现"选择性连接"

三态门 = 数字电路的"开关"

• 使能信号为1 → 连接通路

• 使能信号为0 → 断开通路

只有被允许的信号才能上线，其他信号自动"让路"

⚙️ 三态门电路工作原理

✅ 使能E=1（工作模式）

输入I8 → 三态门 → 输出O8
信号正常传输

⛔ 使能E=0（高阻态）

输入I8 → 三态门 → ⚡高阻
输出端完全断开

🔍 高阻态的本质

高阻态（High Impedance）不是0也不是1，而是"断开连接"的状态。
就像插头没插到插座上，电路处于开路状态，阻抗非常高（接近无穷大），
所以叫"高阻态"，符号用Z表示。

💡 实际应用场景

1. 数据总线（Data Bus）
CPU内部有8位、16位、32位数据总线，多个组件共享同一条总线传输数据

2. 寄存器输出控制
R0、R1、R2、R3等寄存器都连到总线上，通过使能信号控制谁输出

3. 三态缓冲器
用于隔离和控制信号传输方向，防止信号冲突

⚠️ 关键要点：
输出端通常会加三态门，而输入端一般不需要！
因为多个输出同时工作会冲突，但多个输入同时接收信号没问题。

💡 关键设计理念

1. 并行计算：所有运算模块同时工作，同时产生结果
2. 三态门选择：通过使能信号(E)控制，只有一个模块的结果输出到总线
3. 操作控制：控制信号决定激活哪个运算模块
4. 黑盒思维：掌握输入输出关系即可，无需深究每个逻辑门细节

🎮 两大运算类型

🔢

1. 算术运算

定义：处理数值大小的运算

• 加法：A + B

• 减法：A - B（通过加法器 + 补码实现）

• 乘法：重复加法

• 除法：重复减法

💡 本质：都通过加法器(ADD)完成！

图4-34中用深色标注了加法器部分

🔀

2. 布尔逻辑运算

定义：处理真/假条件的运算

(1) 与 (AND)：同时满足两个要求

例：有钱与有时间，才会去旅游

(2) 或 (OR)：满足一个即可

例：今天下雨或懒得出门，就不跑步

(3) 非 (NOT)：取反

例：今天是星期天 → 今天非工作日

(4) 异或 (XOR)：二选一

例：拿了A就不能拿B（只能选一个）

⚠️ 真与假的表示

在计算机中，"真"用1表示，"假"用0表示。
逻辑运算不是处理数字大小，而是处理条件是否成立。
